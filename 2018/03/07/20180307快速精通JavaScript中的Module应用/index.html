快速精通JavaScript中的Module应用

刘宇佳 发表于 2018-03-07

这种模块化的程序设计方案给程序设计提供了一个极大的好处：你可以反复利用自己编写好的代码。

很多时候我们看的书都会分章节，这样不仅作者在写作的时候有条理，读者在读书的时候也能清晰读懂作者的逻辑结构。同样在程序编写领域，也有类似的程序编写风格，一般称之为“Literate Programming”。

在实际编写程序的过程中，一般都用不着这样的编写风格，但是这种模块化的程序设计方案确给程序设计提供了一个极大的好处：你可以反复利用自己编写好的代码。一般资深的程序员都会将不同功能的模块放到各种各样的文件中，当用到的时候他们就直接将模块粘到他们现有的程序中，修改再使用。JavaScript有天然的模块化优势，于是慢慢的演化出了在线的模块库——NPM。

在JavaScript编程过程中，一个号的模块库就是给编写者提供了一个好的接口，能极大地提高开发者的效率。那到底如何在JavaScript中创造模块呢？

在JavaScript中函数是唯一创建域的方式，创建模块也必须基于函数来创建。常规的函数操作中，先定义一个变量，然后定义函数对变量操作；而模块的操作则是将函数直接作为一个对象赋值给一个变量，进而使用时直接调用变量就可以完成函数的所有操作。
