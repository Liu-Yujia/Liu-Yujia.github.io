工程师界的打卡神器Git和GitHub

刘宇佳 发表于 2018-02-26

Git和GitHub是用来备份自己写出来的代码的工具，这是最主要也是最重要的功能。但是仅仅这一点重要的功能自然不足以满足广大码农们以及广大互联网公司们的各种需求，Git和GitHub还可以让码农随时地查看已经提交过的任何版本任何时候的代码，除了自己的代码还可以随意地下载和查看别人公开的代码；对于企业和公司，Git和GitHub可以实现多人的协同工作，极大地增加了工作效率。

当初在第一次学习编程的时候就知道了GitHub，还注册过账号，可当时编程并没有学会，这个东西也看得一塌糊涂，只是往里面胡乱上传了一些东西，根本没有做深入的了解。对任何事都浅尝辄止的状态也客观地反映了我当时的学习和思考是有多么不靠谱，这其实是对自己不负责任的体现。

对于资深的计算机工程师来说，Git和GitHub是每天都必须用的一项工具，简而言之，它们是用来备份自己写出来的代码的工具，这是最主要也是最重要的功能。但是仅仅这一点重要的功能自然不足以满足广大码农们以及广大互联网公司们的各种需求，Git和GitHub可以让码农随时地查看已经提交过的任何版本任何时候的代码，除了自己的代码还可以随意地下载和查看别人公开的代码；对于企业和公司，Git和GitHub可以实现多人的协同工作，极大地增加了工作效率。

但是对于新手来说，Git和GitHub是非常难理解的东西，因为之前从来没用过类似的工具，也从没接触过类似的工作原理。别说新手了，现在很多资深的码农甚至都不知道这个东西。

我之前刚开始接触的时候也是一脸懵逼的状态，Git官方的文档看不懂，网上的讲解视频都是讲得乱七八糟。当时开始编代码一阵子了，急需这样的工具来备份自己的代码，可死活就是学不会，反复地啃了三遍Git官方文档的前两章之后才开始慢慢地用Git和GitHub。现在我还只是会用Git最简单的功能备份代码，可仅仅是这用这么一点简单的功能，就已经让我停不下来了！而对于新手来说，学会备份自己的代码就已经是足够用了，

GitHub的神奇之处

就像文章题目中提到的，GitHub在我看来更像是一个打卡神器，每天你把你写过的代码备份到GitHub上后，你的主页上都会有一个绿色的格子，而绿色格子的颜色深度代表了你写的代码的多少。对于初学者来说，这是一个无法欺骗的大概神器，你要是有一天没有编代码或者偏代码少了，效果就在会在你的主页上呈现出来。

据说在很多互联网公司面试码农的时候，面试官很多时候都会直接打开面试者的GitHub的主页去看看过去一年里编的代码量有多少。所以平时要是不好好地编写，到面试的关头上自然会因为这个吃亏。

另外GitHub是一个社交神器，所有的计算机工程师几乎都会将自己的代码公开，而所有的工程师都可以拷贝修改彼此的代码。这对于工程师们来说简直就是一个非常大的福利，遇到困难时可以在那上面寻求工程师的帮助，还可以免费的围观和下载大神们的神作。自从有了GitHub，不知道多少码农们因此省了宝贵的时间去处理难度更高的问题。

Git的神奇之处

如果说GitHub给工程师们呈现出了最直观的劳动成果，那么Git就是这些劳动成果和运行机制的“最强大脑”。Git是这套工具的核心架构，也是工程师们提交代码时用到的命令和工具。

Git的核心原理和神奇之处一直半会儿还真是说不完，等以后有机会了再系统地谈谈Git。对于初学者来说，几个简易的Git命令和原理就能满足编码需求了。

简易教程

首先要学会配置自己的账号，也是对应的把代码备份到自己的账号上，每次在使用Git之前配置一次，命令为如下两个：

git config –global user.name “XXX”
git config –global user.email XXXXXX@XXX.com

填上自己的用户名和账号，就自动给你识别号账户了。接下来的一个命令看似没什么实质性的用处，但是经常用来查看一下当前的状态：

git status

每次到了不清楚该干什么或者干了什么的时候，就用这个命令来看一下。接着就是真正工作要用的了，首先将自己在GitHub上创建好的仓库clone下来：

git clone github.com/yourname/XXX

(真实用的时候前面还要加上https://)，clone下来之后，你的计算机本地就有一个相应的XXX文件了。把你编好的代码放到这个文件中，然后再到terminal中执行如下命令：

git add .
git git commit -m “new commit”
git push
