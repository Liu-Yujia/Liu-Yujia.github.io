JavaScript学习笔记七

刘宇佳 发表于 2018-01-14

JavaScript学习笔记七

前段时间学习对象和函数的时候，我本以为我学习JavaScript到了一个瓶颈期了，现在才知道，我当时太天真了，完全不知道零基础学习编程的难度所在，学编程越到后面“瓶颈”越多。

最近在看Eloquent JavaScript这本书时，很多时候感觉实在难的看不下去了；有时候看得下去内容，但是读不懂代码；还有时候照着例题敲了半天代码，敲了半天还不理解是什么意思。连续的蒙逼和挫败感就是这么产生的，但能怎么办，既然选择了这条路，那只能义无反顾走到头。

看原版书看得烦躁的时候，我也上网去翻了翻现有的JavaScript免费中文教程，有廖雪峰的JavaScript全栈教程，也有阮一峰的JavaScript 标准参考教程，都大致地浏览了一下。

国内的JavaScript教程还是更加初级一点，作者们都是资深的互联网从业者，教程中的例子和解释也很到位。相对来说，选择Eloquent JavaScript这本书确实是难了一点，书中刚开始的几章内容就涉及到了复杂的函数以及对象代码，例子更是晦涩难懂。我想作者当初在编写这本书的时候，一定是假设了学习这本书的人看完前三四章就掌握一定的编程基础知识，所以第五第六章开始就植入了有相当难度的例题，而且对例题的解释并不是那么透彻。

难归难，Eloquent JavaScript这本书在知识的系统性上还是碾压网上一般的学习教程，这毕竟是一本经过审核校对再版过两次的书，而且在编程界口碑极好。我跟着学了两个月后，感觉再看别的教程就看不下去。究其原因，一是作者的写作思路不一样，跟一个思路学一段时间，再换一种思路学，自己的脑子容易混乱；二是自己的学的还是太浅薄，要是有个JavaScript大神来看这些教程，不管作者的思路是什么样，大神们肯定都能理解。

这几天我一直还在纠缠于之前没学透彻的函数和对象，重新看了书又对部分的例子跟着编译了一遍，可脑子里对于函数和对象的处理和使用还是一塌糊涂。索性也就放了放，开始学习接下来的内容，这两三天的时间里看了本书中的第九章和第十章，关于JavaScript中的一些常规表达和模块化处理。

说实话，看了第九章Regular Expression的内容，并跟着例题敲了一篇代码之后，感觉很多东西都记不住。涉及到的常规表达及用法实在太多了，仅仅几个例题还不足以将这些常规表达消化掉。难怪学编程的人经常要整理cheatsheet，就是为了方便在忘了的时候快速检索到语言中的一些用法，来节约时间。

第十章的Modules倒是个非常意思的章节，主要是将怎么把JavaScript中的程序编译为程序块。对于从事编程工作的人员来说，这个是非常有用的功能，平时的工作中可以积累常用的一些程序块，在用到的时候把这些程序块粘贴到需要编译的程序中稍加改动，就可以直接用这个程序块了。显然这是个非常提升效率的功能。

创建module时，核心的理念还是通过函数来构建一个可调用的程序块，不过表达上跟之前的函数有点不一样。

(function(){
function square(x){
return x*x;
}
var hundred =100;
console.log(square(hundred));
})();

这是一段示例代码，用来计算某个数的平方。可以看到，函数的主题被括号括起来了，在括号的后面还跟了一对括号。如果不跟后面这个括号，编译的过程可以执行，但是不会出结果。这也是JavaScript语言中比较特色的一个点，后面的括号是强制输出结果必要的表达。

当然，使用module提升效率是建立在函数使用非常熟练的基础上，不会编写函数，知道怎么用模块也是白搭。

这章还有很多内容和例题我都还没看，包括十章后面的例题，争取早点把这块内容看完，再次带来一点干货的学习笔记。
